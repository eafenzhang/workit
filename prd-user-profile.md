# PRD：用户信息与 Agent 配置

## 项目信息

- **项目名称**：workit_user_profile
- **编程语言**：TypeScript (Electron + React + Vite + sql.js)
- **原始需求**：为 Workit 桌面智能体工作台建立用户体系——首次登录设置昵称和角色，每个用户即 Agent，可配置人格与记忆技能，角色自动生成头像和 Agent 配置，侧边栏点击用户 icon 进入用户信息页可修改或重置。

---

## 1. 产品目标

| # | 目标 | 衡量标准 |
|---|------|---------|
| G1 | **建立用户身份体系** | 每个 Workit 实例有一个唯一用户身份（昵称+角色+头像），首次启动通过向导完成设置，后续启动自动识别 |
| G2 | **用户即 Agent** | 每个用户的角色决定其 Agent 配置（人格 Prompt、技能集、头像风格），用户感知到"我的 Agent 懂我" |
| G3 | **低摩擦配置与管理** | 侧边栏一键进入用户信息页，2 步内完成修改或重置，无需离开主界面 |

---

## 2. 用户故事

| # | 用户故事 |
|---|---------|
| US1 | **作为新用户**，首次打开 Workit 时，我通过一个 3 步向导（输入昵称 → 选择角色 → 预览确认）完成设置，系统自动为我生成匹配角色的头像和 Agent 配置，让我立刻可以开始使用。 |
| US2 | **作为产品经理**，我选择"产品"角色后，我的 Agent 自动具备产品思维的人格设定和需求分析相关的技能偏好，无需手动调参。 |
| US3 | **作为开发人员**，我点击侧边栏底部的用户头像，进入用户信息页查看我的昵称、角色和当前 Agent 配置，并可以随时修改昵称或切换角色。 |
| US4 | **作为测试人员**，当我切换角色从"测试"改为"研发"时，系统自动重新生成头像和 Agent 配置，无需手动逐项调整。 |

## 3. 功能需求池

### P0 — 必须实现（核心闭环）

| 编号 | 需求 | 说明 |
|------|------|------|
| P0-1 | **首次登录向导** | 首次启动（localStorage 无 user_profile）时展示 3 步向导：Step1 输入昵称 → Step2 选择角色卡片 → Step3 预览头像+昵称+角色+Agent 配置摘要，点击确认保存 |
| P0-2 | **角色预设系统** | 内置 5 种角色（市场/产品/研发/测试/技术），每种角色关联：默认 Agent 人格 Prompt、推荐技能集、头像生成策略（颜色+图标） |
| P0-3 | **头像自动生成** | 根据角色+昵称自动生成头像：使用角色对应颜色作为背景，角色对应 SVG 图标 + 昵称首字叠加，确保视觉区分度 |
| P0-4 | **Agent 配置自动生成** | 选择角色后自动填充该角色的 Agent 人格描述和技能列表，保存到 user_profile |
| P0-5 | **用户信息页** | 侧边栏底部用户 icon 点击进入用户信息页（替换当前无响应状态），展示头像+昵称+角色标签+人格编辑区+技能编辑区+修改/重置按钮 |
| P0-6 | **数据持久化** | user_profile 存储到 localStorage（单用户本地场景），AuthContext 升级为读取 user_profile |

### P1 — 应该实现（完善体验）

| 编号 | 需求 | 说明 |
|------|------|------|
| P1-1 | **角色切换自动重配** | 切换角色后自动重新生成头像和 Agent 默认配置，同时保留用户手动编辑过的人格/技能内容（若未编辑过则覆盖） |
| P1-2 | **头像自定义上传** | 在用户信息页支持上传自定义头像图片（本地文件选择），替代自动生成头像 |
| P1-3 | **重置功能** | "重置为默认"按钮：将当前角色的 Agent 配置恢复为预设值，头像恢复为自动生成 |
| P1-4 | **昵称校验** | 昵称 1-20 字符，不允许为空或纯空格，实时校验提示 |

### P2 — 锦上添花

| 编号 | 需求 | 说明 |
|------|------|------|
| P2-1 | **个性化记忆技能** | Agent 技能支持自定义添加/删除，用户可扩展默认技能集（如添加"代码审查"技能） |
| P2-2 | **角色描述动画** | 向导 Step2 选择角色卡片时展示角色描述和动画过渡效果 |
| P2-3 | **多用户切换** | 支持添加多个用户 profile，启动时可选择以哪个身份登录（目前 P0 仅单用户） |

## 5. 数据结构设计

### 5.1 user_profile 表（localStorage 存储）

```typescript
interface UserProfile {
  // ── 基础信息 ──
  nickname: string;           // 昵称（1-20字符）
  role: RoleType;             // 角色枚举
  avatarType: 'auto' | 'custom'; // 头像类型
  avatarData?: string;        // 自定义头像 base64 / data URL（avatarType='custom' 时）
  avatarColor: string;        // 自动生成头像的背景色（hex）
  avatarIcon: string;         // 自动生成头像的 SVG 图标名

  // ── Agent 配置 ──
  agentPersonality: string;   // Agent 人格描述（system prompt）
  agentSkills: string[];      // Agent 技能列表
  customSkills: string[];     // 用户自定义添加的技能（P2-1）

  // ── 元数据 ──
  createdAt: string;          // ISO 8601 创建时间
  updatedAt: string;          // ISO 8601 最后修改时间
  isFirstTime: boolean;       // 是否已完成首次设置向导
}

type RoleType = 'marketing' | 'product' | 'developer' | 'tester' | 'tech';
```

### 5.3 头像生成规则

```
avatarColor = ROLE_PRESETS[role].color
avatarIcon  = 角色对应 SVG 图标（纯白，居中叠加在彩色圆形背景上）
昵称首字   = 取 nickname[0]，白色粗体，显示在头像右下角小圆内（可选）

头像尺寸: 40px (侧边栏) / 96px (用户信息页)
```

### 5.4 与现有 AuthContext 的兼容

当前 `AuthContext` 的 `User` 接口（id, phone, nickname, avatar, role）改为从 `UserProfile` 派生：

```typescript
// 兼容层：从 UserProfile 构建旧 User 接口
function toLegacyUser(profile: UserProfile): User {
  return {
    id: 1,                              // 单用户固定 ID
    phone: '',                          // 本地无手机号
    nickname: profile.nickname,
    avatar: profile.avatarData || '',   // 自定义头像或空（自动生成头像由组件渲染）
    role: profile.role,
  };
}
```

## 7. Open Questions

| # | 问题 | 决策建议 |
|---|------|---------|
| Q1 | user_profile 是否需要同步到后端数据库？当前仅存 localStorage | **建议 P0 仅 localStorage**。单用户桌面应用场景，localStorage 足够。后续若支持多设备同步再加后端持久化。 |
| Q2 | Agent 人格和技能是否要在每次对话时作为 system prompt 注入？ | **建议由调用方决定**。user_profile 仅存储配置，AI 调用时由各功能模块读取并选择性地注入。短期在设置页展示即可。 |
| Q3 | 自定义头像上传后是否需要裁剪？ | **建议 P1 仅上传+缩略**。上传后前端自动缩放到 200x200px 并转为 base64 存储，暂不做交互式裁剪。 |
| Q4 | 角色切换后，之前手动修改的 Agent 人格是否保留？ | **建议增加"脏标记"**。若用户从未手动编辑过人格/技能（dirty=false），切换角色时自动覆盖；若已手动编辑（dirty=true），弹出确认框询问是否覆盖。 |
| Q5 | 昵称首字生成头像时，多语言字符如何处理？ | **建议取 Unicode 第一个完整字符**。中文取第一个汉字，英文取首字母大写，emoji 直接渲染。无法渲染时 fallback 到角色图标。 |
