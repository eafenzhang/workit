# PRD：Workit 首页重设计 & 洞察页整合

> 版本 v1.0 | 产品经理：许清楚 | 日期：2025-05-31

---

## 一、项目信息

| 项目 | 内容 |
|------|------|
| **语言** | 中文（用户界面中文） |
| **技术栈** | Vite + React 19 + Tailwind CSS + Recharts + Lucide React |
| **项目名称** | `workit` |
| **受影响模块** | `Index.tsx`、`Sidebar.tsx`、`Dashboard.tsx`、`Insights.tsx`、后端 API |

---

## 二、产品目标

1. **降低使用门槛**：新用户进入应用第一眼看到对话式首页，可自然用自然语言表达需求（如"帮我分析上周采集数据"），而非面对冰冷的统计面板。
2. **信息架构精简**：消除"仪表盘"与"洞察分析"的内容重叠，两者各自有 KPI 卡片和图表，合并后形成"统计 → 洞察 → 时间线"自上而下的统一分析视图，减少认知负担。
3. **保留快捷入口**：在对话式首页中保留采集库、知识库等核心功能的快捷入口，确保老用户的肌肉记忆不丢失。

---

## 三、用户故事

1. **As a 新用户**，I want 打开应用后看到一个友好的对话式首页（欢迎语 + 输入框），so that 我能快速理解产品能做什么，并自然发起第一个任务。
2. **As a 日常用户**，I want 在首页直接输入自然语言需求（如"创建一个关于搜索优化的需求"），so that 我可以跳过导航菜单，直达任务。
3. **As a 数据分析用户**，I want 在洞察分析页一眼看到 KPI 统计卡片、AI 洞察报告、最近活动时间线，so that 我能快速掌握项目整体状况。
4. **As a 老用户**，I want 侧边栏导航保留我熟悉的所有入口（采集库、知识库等），只是仪表盘被首页替代，so that 我的工作流不受影响。
5. **As a 产品负责人**，I want 原仪表盘的后端 API 切换为洞察分析页的数据源，so that 不产生冗余接口和数据不一致。

## 四、功能需求池

### P0（必须实现）

| 编号 | 需求 | 说明 |
|------|------|------|
| P0-1 | **新增 Home 页面** | 居中对话式布局，包含欢迎语、输入框、历史对话列表。输入框支持 Enter 提交、Shift+Enter 换行。 |
| P0-2 | **Home 设为默认标签页** | 启动时自动打开 Home 标签页，替代原 `dashboard` 为默认。`Index.tsx` 中 `initial tabs` 改为 `[{ id: 'home', title: '首页', type: 'home' }]`。 |
| P0-3 | **侧边栏新增「首页」入口** | `navItems` 数组最顶部新增 `{ id: 'home', label: '首页', icon: HomeIcon }`。删除 `dashboard` 导航项。 |
| P0-4 | **删除 Dashboard 页面** | 移除 `Dashboard.tsx`、相关 lazy import、路由 case、`MENU_MAP` 中的 dashboard 条目。确保无残留引用。 |
| P0-5 | **Dashboard 统计卡片迁移至 Insights 顶部** | 原 Dashboard 的 4 张 StatCard（总需求、进行中、已完成、活跃用户等）迁移到 Insights 页顶部，放在标题下方、图表上方。数据从 `/api/dashboard/stats` 改为 `/api/insights/kpis` 扩展返回。 |

| P1-1 | **首页欢迎语动态化** | 根据时间段显示不同欢迎语（上午好/下午好/晚上好），显示用户名称（从 Profile 获取）。 |

## 六、数据结构

### 6.1 首页对话消息

```typescript
interface HomeMessage {
  id: string;                    // UUID
  role: 'user' | 'assistant';   // 发送方
  content: string;               // 消息内容（支持 Markdown）
  timestamp: number;             // Unix ms
}
```

### 6.2 首页快捷入口

```typescript
interface QuickAction {
  id: string;           // 唯一标识
  label: string;        // 显示名称，如"采集需求"
  icon: string;         // Lucide 图标名，如 'SparklesIcon'
  action: string;       // 点击行为类型：'navigate' | 'prompt'
  target?: string;      // action=navigate 时目标 tab type
  prompt?: string;      // action=prompt 时预填充到输入框的文本
}
```

### 6.3 首页建议 prompt（静态配置）

```typescript
const SUGGESTED_PROMPTS: string[] = [
  '本周需求统计',
  '分析采集趋势',
  '创建一个技术需求',
  '查看我的待处理任务',
];
```

### 6.4 Insights KPI（扩展现有结构）

```typescript
interface InsightKPI {
  label: string;          // 如"总需求"
  value: string;          // 如"1,234"
  change: string;         // 如"+12%"
  up: boolean;            // 涨跌方向
  icon: string;           // Lucide 图标名
  color: string;          // 图标颜色
  bg: string;             // 图标背景色
}
```

### 6.5 Insights 活动条目

```typescript
interface ActivityItem {
  id: number;
  icon: string;
  color: string;
  text: string;
  time: string;
  targetType?: string;     // 新增：跳转类型 'requirement' | 'knowledge' | 'insight'

```

### 6.6 后端 API 变更

| 原接口 | 变更 | 新用途 |
|--------|------|--------|
| `GET /api/dashboard/stats` | **删除**，数据合并到 | `GET /api/insights/kpis`（扩展返回 4 项 KPI） |
| `GET /api/dashboard/activities` | **删除**，数据合并到 | `GET /api/insights/activities?limit=20`（新增） |
| `GET /api/dashboard/charts` | **删除**，数据合并到 | `GET /api/insights/charts`（扩展：areaData + barData + pieData） |

| `GET /api/insights/charts` | 扩展返回 | 增加 areaData 活动趋势数据 |
